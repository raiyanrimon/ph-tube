Discuss the scope of `var`, `let`, and `const`

var:

  1. Function Scope : it can be accessed throughout the Function If declared in any function.
  2. hoisting: variables declared by 'var' are hoisted to the top of the function or global scope.
  3. we can redeclare and reassign it

let:

  1. Block scope / local scope: they are limited in '{}' , If they are defined in it. we can't access this out of the scope.
  2. not hoisted: it is not hoisted the variables.
  3. we can reassign it but can't redeclare.

const:

  1. Block scope / local scope: they are limited in '{}' , If they are defined in it. we can't access this out of the scope.
  2. not hoisted: it is not hoisted the variables.
  3. we can't redeclare or reassign it and it is immutable.

Tell us the use cases of `null` and `undefined

Null:
  1. If variable's value is empty or has no value, then we can use null
  2. clearing or reset the value
  3. we can use it as placeholder
  4. we can use it to compare
Undefined:
  1. If a variable is declared but value is not defined
  2. If function is without return or return without value
  3. If a function's parameter is not defined
  4. If object has no property
  5. If any element of array is deleted
  6. If value is set as undefined
